# KISA ICT 중소기업 정보보호 지원사업 Rentme 적합성 판단

작성일: 2026-07-15

## 결론

조건부 Go로 판단한다. Rentme는 세입자/임대인 신뢰 정보, 증빙 서류, 연락처 공개 여부, 접근 로그를 다루므로 KISA 정보보호 컨설팅 및 보안 패키지 지원 목적과 잘 맞는다. 다만 신청 주체가 `중소기업`으로 인정되는지, 사업장 소재지 기준 권역센터가 어디인지, 보안제품/SECaaS 도입 비용의 수요기업 부담금 20%를 승인할 수 있는지 확인되기 전에는 제출하지 않는다.

## 공식 공고 확인

- KISA 공고: 2026-06-26 등록, `2026년 ICT 중소기업 정보보호 지원 사업 수요기업 모집 공고`
- 사업 목적: 침해사고 피해 지역 중소기업, 지역 전략산업 영위 중소기업 등을 대상으로 정보보호 컨설팅, IT 보안 패키지, SECaaS 패키지 이용을 지원
- 지역정보보호센터 자료실: 2026-07-08에 수도권/충청권, 호남권, 영남권 수요기업 지원 가이드 게시
- 신청 채널: KISA 지역정보보호센터 `https://risc.kisa.or.kr/`

## Rentme 적합성

적합하다.

- 개인정보 처리 흐름: 회원, 프로필, 임대 신뢰 리포트, 추천인/레퍼런스, 서류 업로드 기능이 있어 개인정보보호 컨설팅 대상이 명확하다.
- 접근통제/열람 로그: `lib/consent-access.ts`와 `app/api/access-logs/route.ts`에서 동의 기반 열람, 허용 필드, 열람 결과, IP, user-agent를 기록한다.
- 관리자 감사: `lib/admin.ts`와 `admin_logs` 테이블이 관리자 문서 심사 및 신뢰점수 조정 로그를 남긴다.
- 출시 전 보안 점검성: `docs/LAUNCH_CHECKLIST.md`와 `scripts/launch-smoke.mjs`에 운영 비밀값, admin route protection, launch smoke 흐름이 있어 컨설팅 산출물과 연결하기 쉽다.

## 신청 전 보드 확인값

- 신청 주체: 법인/개인사업자/예비창업 상태와 `중소기업기본법`상 중소기업 해당 여부
- 사업장 소재지: 지역정보보호센터 권역 선택 기준. 현 후보는 수도권/충청권 가이드 기준으로 보되, 실제 사업장 주소에 따라 변경한다.
- 예산 승인: 보안제품 또는 SECaaS 도입 시 정부지원금 80%, 수요기업 부담금 20% 및 VAT 부담 승인 여부
- 신청 범위: 정보보호 컨설팅만 우선 신청할지, 컨설팅 결과 기반 보안 패키지/SECaaS까지 신청할지
- 제출 권한: KISA 지역정보보호센터 계정, 사업자등록증/중소기업 확인 증빙, 담당자 연락처
- 제출 자료 제한: 실제 사용자 개인정보, 원본 증빙 이미지, 실서비스 secret, 내부 DB 접속정보는 제출 제외

## 권장 신청 패키지

1. 정보보호 컨설팅 우선
2. 결과에 따라 IT 보안 패키지 또는 SECaaS 선택
3. Rentme 점검 범위는 개인정보 처리흐름, 동의/철회, 접근 로그, 관리자 권한, 파일 업로드/보관, 운영 secret 관리로 제한

## No-Go 조건

- 신청 주체가 중소기업 또는 사업자 요건을 충족하지 못한다.
- 보드가 20% 부담금 및 VAT 예산을 승인하지 않는다.
- 실제 사용자 개인정보 또는 민감 증빙 제출이 요구되고, 비식별/샘플 자료로 대체할 수 없다.
- 보안 컨설팅보다 출시 필수 결함 수정이 우선인 상태로 판명된다.
