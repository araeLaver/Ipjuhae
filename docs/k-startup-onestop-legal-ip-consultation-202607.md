- 브랜드명: `Rentme(입주해)` 병기
- 임대 적합 참고 지표 산식: 전체 코드, 세부 배점, `trust.score_breakdown`, 0~100 점수 전제는 제출하지 않고 기능 수준의 산출 취지만 설명
- 예시 데이터: 실제 사용자 데이터 없이 가상 세입자/집주인 예시만 제출

- 첨부 범위: 실제 개인정보 없는 demo 화면 3~5장 또는 정책 조항 발췌만 제출할지

## 상담 신청 요약

Rentme는 세입자가 본인의 임대 확인 항목 리포트를 만들고, 집주인이 동의 기반으로 세입자 정보를 열람해 매칭을 검토하는 주거 임대 매칭 서비스입니다. 현재 MVP에는 이름 마스킹, 재직/소득/신용 관련 확인 배지, 이전 집주인 레퍼런스, 서비스 내부 참고 지표, 집주인용 세입자 검색/필터, 동의 기반 리포트 접근 로그 기능이 포함되어 있습니다.

출시 전 다음 사항에 대해 법률 및 특허·지재권 자문을 받고자 합니다.

1. 세입자의 재직, 소득, 신용등급, 생활 패턴, 이전 집주인 레퍼런스를 수집·가공·공유할 때 필요한 동의 항목과 개인정보처리방침 기재 수준
2. `신용 인증`, `검증된 세입자`, `높은 신뢰 점수 = 더 좋은 조건`처럼 개인신용평가, 신용정보업, 계약조건 보장 또는 임차인 차별 도구로 오인될 수 있는 표현의 대체 필요성
3. 집주인이 내부 참고 지표, 확인 여부, 흡연/반려동물/가족 유형 등으로 세입자를 필터링하는 기능의 공정성·차별 리스크
4. Rentme가 세입자와 집주인을 연결하고 메시지를 제공하는 구조가 공인중개사법상 중개행위 또는 중개대상물 광고 규제에 해당할 가능성
5. `Rentme`, `입주해`, `임대 적합 참고 지표`, `확인 항목 리포트` 등 명칭의 상표 출원 가능성과 서비스 설명 문구의 IP 보호 전략
6. 리포트 상세 항목, 검증값, 확인 필요 항목, 요약 신호를 집주인·세입자·내부 운영자에게 각각 어느 수준까지 공개해도 되는지

## 제품 기능 요약

- 세입자 프로필: 이름, 연령대, 가족 유형, 반려동물, 흡연 여부, 거주 패턴, 희망 지역, 자기소개
- 인증/검증: 재직 인증, 소득 인증, 신용 인증, 서류 업로드 및 검증값
- 레퍼런스: 이전 집주인에게 설문을 요청하고 완료된 응답을 점수 계산에 반영
- 서비스 내부 참고 지표: 프로필 완성, 재직/소득/신용 관련 확인, 레퍼런스, 서류 검증, 분쟁/정정 요청, 주거 안전 보정 등을 참고해 내부적으로 산출하는 요약 신호. 외부 상담 자료에는 `trust.score_breakdown`, 세부 산식, 범주별 배점, 0~100 점수 전제, 현재 구현 최대값을 제출하지 않음
- 집주인 검색: 지역, 가족 유형, 반려동물, 흡연 여부, 소음 수준, 거주 기간, 확인 항목 요약, 확인 배지, 레퍼런스 보유 여부로 필터링
- 동의/접근통제: `consents` 기반으로 열람자, 목적, 허용 필드, 연락처 공개 여부, 유효기간, 접근 로그를 관리하는 방향으로 구현 중

## 핵심 상담 질문 6개

### 1. 개인정보 동의 및 제3자 제공

Rentme가 세입자의 재직 회사명, 소득 구간, 신용등급, 서류 이미지, 이전 집주인 레퍼런스, 생활 패턴 정보를 수집하고 집주인에게 일부 공개하려면 어떤 단위로 동의를 받아야 하는가?

- 회원가입 필수 동의와 프로필 공개/집주인 열람 동의를 분리해야 하는지
- `재직`, `소득`, `신용`, `레퍼런스`, `연락처`를 별도 선택 동의로 나누어야 하는지
- 집주인 1명/매물 1건/기간 1회 등으로 접근권을 제한해야 하는지
- 동의 철회 시 이미 열람한 집주인에게 어떤 조치를 고지해야 하는지
- 개인정보처리방침에 접근 로그, 알고리즘 가공, 자동화된 점수 산출을 어느 수준까지 설명해야 하는지

### 2. 신뢰점수와 신용정보업 오인 가능성

Rentme의 `신뢰점수`는 임대차 매칭 참고 지표로 설계되어 있으며 금융거래용 신용평가를 의도하지 않는다. 다만 입력 항목에 `신용 인증`, `소득 인증`, `재직 인증`, 레퍼런스가 포함된다.

- `신뢰점수`, `신용 인증`, `신뢰 프로필` 표현이 개인신용평가, 개인신용정보 처리, 신용정보업으로 오인될 수 있는지
- 서비스 화면에 "금융 신용평가가 아니며 임대차 참고 정보이고, 계약 가능성·납부능력·신용상태를 보증하지 않는다"는 고지를 넣으면 충분한지
- `신용 인증` 대신 "사용자 동의 기반 신용 관련 확인 항목", "신용 관련 서류 확인 여부"처럼 표현을 낮추는 것이 필요한지
- `검증된 세입자`, "높은 신뢰 점수 = 더 좋은 조건"처럼 계약조건 보장으로 읽힐 수 있는 문구를 금지해야 하는지
- 점수 산식의 전체 공개, 범주별 공개, 기능 설명만 공개, 완전 비공개 중 어떤 수준이 적정한지
- 점수에 대한 열람, 정정, 이의제기, 재산정 절차를 제품 기능, 약관, 개인정보처리방침 중 어디까지 넣어야 하는지

### 3. 집주인 필터링의 공정성·차별 리스크

집주인 검색 기능은 신뢰점수, 인증 여부, 가족 유형, 반려동물, 흡연 여부, 소음 수준, 거주 기간 등으로 세입자 후보를 필터링한다.

- 해당 필터가 임차인 차별 또는 부당 배제 수단으로 해석될 수 있는지
- 가족 유형, 반려동물, 흡연, 소음 수준 같은 생활 정보 필터의 허용 범위와 고지 문구
- 신뢰점수 최저값 필터를 제공하는 것이 위험한지
- 집주인 화면에 "최종 계약 여부는 당사자 판단이며 점수/필터는 참고 정보" 문구가 필요한지
- 필터링 결과와 집주인 열람 이력을 기록해야 하는지

### 4. 부동산 중개행위 해당성

Rentme는 매물 광고 중심보다 세입자 프로필 기반 매칭, 메시지, 역제안 UX를 지향한다. 향후 계약 성사 수수료 또는 유료 매칭 기능을 검토할 수 있다.

- 세입자 프로필 검색, 집주인 메시지 발송, 매칭 추천이 공인중개사법상 중개행위에 해당할 수 있는 경계
- 회사가 계약 조건 협상, 계약서 작성, 보증금/월세 조건 제안에 관여하지 않으면 정보제공 플랫폼으로 볼 수 있는지
- "매칭", "역제안", "계약 성사 수수료" 표현 또는 BM이 중개업 등록 필요성을 만들 수 있는지
- 매물 정보가 포함될 경우 중개대상물 인터넷 표시·광고 규정을 따라야 하는지
- 직거래 플랫폼 운영 가이드상 실명인증, 허위매물 모니터링, 신고 처리 의무를 어느 수준으로 준비해야 하는지

### 5. 브랜드/IP 및 상표

대외 브랜드는 `Rentme`와 `입주해`를 병행 검토 중이며, 제품 핵심 개념으로 `신뢰점수`, `신뢰 프로필`, `검증 배지`를 사용한다.

- `Rentme`, `입주해` 상표 출원 가능성 및 선행상표 조사 범위
- `신뢰점수`처럼 설명적인 명칭을 상표로 보호할 수 있는지, 대체 명칭이 필요한지
- 알고리즘 산식, 리포트 형식, 인증 배지 디자인을 영업비밀/저작권/상표/디자인 중 무엇으로 보호할 수 있는지
- 특허·BM 특허 검토 가치가 있는지
- 투자자료와 서비스 화면에서 사용할 수 있는 권리표시 문구

### 6. 리포트 상세 공개 범위와 정정 절차

Rentme는 세입자 동의와 열람 목적에 따라 집주인에게 일부 리포트를 보여주는 구조를 검토 중이다. 원본 서류 대신 검증값, 상태값, 확인 필요 항목, 요약 신호를 제공하는 방향이 최신 특허 명세서와 제품 방향에 가깝다.

- 집주인에게 `scoreBreakdown`, 부정 레퍼런스, 분쟁 페널티, 검증 상세 사유를 보여줘도 되는지
- 원본·상세 사유 대신 "확인 완료", "추가 확인 필요", "요약 신호" 수준으로 제한해야 하는지
- 세입자가 본인 리포트를 열람하고 정정, 이의제기, 재산정을 요청하는 절차를 어느 수준까지 제공해야 하는지
- 집주인, 세입자, 내부 운영자별 리포트 공개 범위를 목적별로 다르게 설계해야 하는지
- `신뢰점수` 대신 `신뢰 프로필 요약`, `임대 적합 참고 지표`, `확인 항목 리포트` 같은 대체 명칭을 병행하거나 전환하는 것이 필요한지

## 상담 후 반영 후보 문구

아래 문구는 법률 검토 전 초안이다.

- 신뢰점수 고지: "Rentme 신뢰점수는 세입자가 제공하고 검증을 요청한 임대 관련 정보를 바탕으로 산출한 서비스 내부 참고 지표입니다. 금융 신용평가, 채무상환능력 평가, 임대차 계약 체결 보증을 의미하지 않습니다."
- 인증 배지 고지: "인증 배지는 제출 자료 또는 연동 기관을 통해 특정 항목의 확인 절차가 완료되었음을 의미하며, 정보의 영구적 정확성이나 계약 이행을 보증하지 않습니다."
- 집주인 화면 고지: "검색 필터와 점수는 후보 탐색을 돕기 위한 참고 정보입니다. 계약 여부는 관계 법령과 공정한 기준에 따라 당사자가 직접 판단해야 합니다."
- 동의 고지: "세입자는 공개 범위, 열람 목적, 열람 가능 기간을 선택할 수 있으며, 동의를 철회하면 이후 열람은 제한됩니다."
- 중개행위 고지: "Rentme는 당사자 간 정보 탐색과 커뮤니케이션을 지원하며, 별도 고지 없이 임대차 계약 체결을 대리하거나 중개하지 않습니다."

## 상담 후 반영 체크리스트

- 개인정보처리방침: 수집 항목을 필수/선택/집주인 열람 공개 항목으로 분리하고, 제3자 제공 표를 추가한다.
- 개인정보처리방침: 신뢰점수 산출, 자동화된 가공, 접근 로그, 동의 철회 후 처리 방식을 명시한다.
- 이용약관: "중개자" 표현을 "정보 탐색 및 커뮤니케이션 지원자" 또는 상담 결과에 따른 대체 표현으로 바꾼다.
- 이용약관: 신뢰점수와 인증 배지가 계약 체결, 임대료 납부, 신용상태를 보증하지 않는다는 면책 문구를 추가한다.
- UI: `신용 인증` 문구를 `신용 관련 서류 확인` 또는 상담 결과에 따른 완화 표현으로 교체할지 결정한다.
- UI: 집주인 검색 화면과 세입자 카드에 점수/필터 참고용 고지를 추가한다.
- 제품: 신뢰점수 열람, 정정 요청, 레퍼런스 이의제기, 재산정 요청 흐름을 정책과 기능으로 연결한다.
- 제품: 집주인에게 상세 산식·부정 레퍼런스·분쟁 페널티를 직접 노출하지 않고, 확인 상태·추가 확인 필요·요약 신호 중심으로 리포트를 재구성할지 결정한다.
- 대외자료: `신뢰점수`, `Trust Score`, `신용조회`, `검증된 세입자`, "높은 신뢰 점수 = 더 좋은 조건"류 문구와 세부 산식 공개를 제거하거나 완화한다.
- 운영: 상담 답변 원문, 내부 결정, 반영 PR/커밋을 출시 체크리스트에 링크한다.
