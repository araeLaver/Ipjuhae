# 공공데이터 활용기업 AI연계 맞춤형 성장 지원사업 Rentme 적합성 판단

작성일: 2026-07-17
최종 확인: 2026-07-18

## 결론

판단: 조건부 Go

2026년 공공데이터 활용기업 AI연계 맞춤형 성장 지원사업은 Rentme의 7월 2순위 제출 후보로 적합하다. Rentme는 임대차 정보 요약 리포트와 지역 생활권 요약을 공공데이터 기반 제출용 demo 설계 후보로 제시할 수 있고, 실제 사용자 개인정보 없이 mock profile과 sample data로 제출 범위를 제한할 수 있다.

다만 외부 제출 전에는 신청 주체, 이메일 접수 담당자, 첨부 범위, AI 활용 범위 1순위가 확정되어야 한다. 보드 승인 전에는 이메일 접수나 신청서 발송을 실행하지 않는다.

## Rentme 적합성

적합하다.

- 공공데이터 활용성이 명확하다: 전월세 실거래가, 법정동/행정구역, 교통 접근성, 공공임대주택, 생활 인프라 데이터를 결합해 임대 매물 주변의 객관 정보를 설명할 수 있다.
- AI 적용 범위를 좁힐 수 있다: 신용평가나 자동 심사 대신 `지역/생활권 요약`, `리포트 설명 문장 생성`, `FAQ 응답`, `운영 분석`으로 제한하면 규제 리스크가 낮다.
- 개인정보 없는 제출이 가능하다: 실제 임차인·임대인 데이터 대신 sample listing, synthetic profile, 공개 공공데이터 row, mock 화면으로 과제를 설명할 수 있다.
- 사업 단계와 맞는다: K-DATA 문제정의형 지원보다 마감 여유가 있고, 출시 전 공공데이터 기반 설명 가능성을 정리하는 2순위 과제로 적합하다.

## 권장 문제정의

`동의 기반 임대 정보 요약 리포트 + 공공 생활권 설명 AI`

Rentme는 사용자가 동의한 범위의 프로필 요약과 공개 공공데이터를 분리 저장·표시하는 구조를 제안하고, AI 활용은 공개 데이터와 비식별 요약만 사용해 지역·거래·생활권 맥락을 이해하기 위한 설명 문장을 생성하는 범위로 제한한다. 핵심은 "세입자/임대인을 평가하는 점수"가 아니라 "공개 데이터 기반의 지역·거래·생활권 맥락을 이해하기 쉽게 설명하는 리포트"로 포지셔닝한다.

## 사용자 동의 데이터와 공공데이터 분리 원칙

- 저장 분리: 사용자 동의 데이터는 계정·프로필·문서·열람 로그 영역에 보관하고, 공공데이터는 지역/법정동/좌표 기준 reference table 또는 cache로 분리한다.
- 처리 분리: AI prompt에는 실제 주민등록번호, 계좌, 원본 증빙, 연락처, 상세 주소를 넣지 않는다. 필요 시 동의된 필드의 비식별 요약만 넣는다.
- 표시 분리: 리포트 화면에서 `공공데이터 기반 설명`과 `사용자 동의 기반 요약`을 라벨로 분리한다.
- 권한 분리: 공공데이터 조회는 서비스 공통 권한으로 처리하고, 사용자 데이터 열람은 동의 범위와 access log를 남기는 흐름만 허용한다.
- 제출 분리: 지원사업 신청서에는 실제 사용자 개인정보, 원본 증빙, production secret, 내부 DB dump를 첨부하지 않는다. mock 화면과 sample schema로 대체한다.

## 권장 AI 활용 범위

1순위는 `리포트 설명 및 지역/생활권 요약`이다.

- 입력: 공공데이터 row, 지역 통계, mock 매물 정보, 제출용 synthetic 비식별 프로필 요약
- 출력: "이 지역의 전월세 거래 범위", "교통/보육/공공임대 인프라", "사용자가 직접 확인해야 할 항목"을 설명하는 문장
- 제외: 신용점수 산정, 임차인 자동 합격/탈락, 임대인 위험도 판정, 법률 조언, 개인정보 원문 분석

## 신청 전 보드 확인값

- 신청 주체: 예비창업자, 개인사업자, 법인 중 무엇으로 제출할지
- 제출 표기명: 신청서에 사용할 팀명/서비스명/대표자명
- 접수 담당자: `opendatabiz@kmac.co.kr`로 발송할 이메일 계정과 담당자 연락처
- 첨부 범위: 신청서, 사업계획 요약, demo 화면, sample schema, 개인정보 없는 mock data까지 제출 가능한지
- AI 활용 범위: `리포트 설명/지역 요약`을 1순위로 고정할지
- 공공데이터 범위: 5~8개 후보 중 신청서에 넣을 최종 데이터셋
- 개인정보 제외 원칙: 실제 사용자 데이터, 원본 증빙, secret, 내부 DB 접속정보 제출 금지 승인
- 최종 제출자: 보드가 최종 검토 후 직접 발송할지, 대리 발송을 승인할지

## 신청 초안 목차

1. 신청기업/팀 기본정보
   - 신청 주체, 대표자, 연락처, 이메일 제출 계정, 서비스명 `Rentme`
2. 서비스 개요
   - 임대차 의사결정 전 공공데이터 기반 지역·거래·생활권 맥락을 설명하는 동의 기반 리포트
3. 해결하려는 문제
   - 임차인과 임대인이 공개 시장정보, 생활권 정보, 동의 기반 프로필 요약을 분리해 확인하기 어려운 문제
4. 활용 공공데이터
   - 전월세 실거래가, 법정동/행정동 코드, 부동산 통계, 공공임대주택, 교통, 보육/생활 인프라 데이터
5. AI 연계 계획
   - 공개 공공데이터와 synthetic profile을 입력으로 지역·거래·생활권 설명 문장을 생성하고, 신용평가·자동 심사·법률 조언은 제외
6. 개인정보·보안 원칙
   - 실제 사용자 개인정보, 원본 증빙, production secret, 내부 DB dump 미제출
7. 제출용 demo 구성
   - mock listing, synthetic profile, sample schema, 공공데이터 기반 리포트 화면, AI 설명 예시
8. 지원 필요 영역
   - AI-Ready 구축, AI서비스 고도화, 데이터 활용 컨설팅, 향후 투자/해외진출 패키지 중 우선순위 선택
9. 기대효과와 일정
   - K-DATA 이후 2순위 제출 후보로 2026-07 말 초안 작성, 2026-08 초 보드 검토, 2026-08-14 전 이메일 제출

## No-Go 조건

- 신청 주체와 접수 담당자가 2026-08-14 전까지 확정되지 않는다.
- 지원사업 신청서가 실제 사용자 개인정보 또는 원본 증빙 제출을 요구한다.
- Rentme를 신용평가, 임차인 자동 심사, 부동산 중개 자동화처럼 설명해야만 한다.
- K-DATA 2026-07-21 마감 대응을 방해할 정도로 신청서 작성량이 커진다.
